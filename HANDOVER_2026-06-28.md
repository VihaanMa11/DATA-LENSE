## 1. What this project is

A white-label MIS/BI dashboard (React + Vite frontend, Express backend) that reads a **Google Sheets workbook** (exported as `.xlsx`) and renders Power-BI-style analytics. Clone the repo, paste a Google Sheet URL in the UI, press **Sync Now**, and the dashboard populates. Stateless, no database.

- **Frontend:** React 19 + Vite, ApexCharts (`react-apexcharts`) for all charts, custom CSS in `src/styles.css`.
- **Backend:** Express (`server/index.js`), `xlsx` for parsing, pure-JS analytics.
- **Auth:** shared-password gate (HMAC cookie). No accounts, no DB.
- **Data:** direct-to-sheet, stateless, in-memory cache keyed by sheet id + content signature.

## 4. NEW FORMAT SPEC (studied from the file — authoritative)

**File:** `MLH_Master_Data_FY2024-27.xlsx` (also the target Google Sheet shape).
**Tabs (12):** `ReadMe`, `CrNote`, `DrNote`, `JournalRegister`, `Receipt`, `Payment`, `PurchaseReturn`, `SalesReturn`, `Purchase`, `Sales`, `ItemMaster`, `AccountMaster`.

**Financial Year column** is the **first column of every data tab.** Values (exact strings):
`"FY 2024-25"`, `"FY 2025-26"`, `"FY 2026-27"`.
Periods: FY24-25 = 01-04-2024→31-03-2025, FY25-26 = 01-04-2025→31-03-2026, FY26-27 = 01-04-2026→15-06-2026 (in progress). DrNote has FY24-25 & FY25-26 only.

### Item registers — `Sales`, `Purchase`, `SalesReturn`, `PurchaseReturn` (same columns)
```
Financial Year | Vch. Series | Bill Date | Bill No. | Party Name | Billed Party Name |
Narration | Item Name | Main Qt | Billed Quantity Alt | List Price | Price | Amount |
Amount Grand Total | Transport | Station | Salesman Name | Vch Creation Date | Vch Creation Time | Created By
```
- `Amount` = line-level amount. `Amount Grand Total` = **bill-level total** (this REPLACES old `Final Amt`; only meaningful on header/first row of a bill).
- `Main Qt` = quantity. `Price` = unit price. `Station` is now **per-line** (was master-only). **`Salesman Name` is NEW** (old data lacked it — salesman analysis is now possible).
- Old format had `Distance` (gone) and `Final Amt` (now `Amount Grand Total`).
- Sales tab ≈ 24,720 rows across 3 FYs.

### Ledger registers — `CrNote`, `DrNote`, `JournalRegister`, `Receipt`, `Payment` (same columns)
```
Financial Year | Bill Date | Bill No. | Account Name | Debit Amount | Credit Amount
```
- Customer collections = **credit** side of `Receipt` (account = customer). Vendor payments/expenses = **debit** side of `Payment` (account = payee).

### `ItemMaster`
```
Financial Year | Name | Alias | Group Name | Main Unit | Alt. Unit | Sale Price | Purc. Price |
MRP | Tax Category | HSN Code | Op. Stock(Main) | Op. Stock(Alt.)
```
- `Op. Stock(Main)` = opening stock. Master is **per-FY** (one row per item per FY).

### `AccountMaster`
```
Financial Year | Name | Group Name | Address Line1..4 | Tel. No. | Mobile No. | E-Mail | Alias |
GST No. | Station | Pincode | State | Opening Bal. (Dr) | Opening Bal. (Cr)
```
- Join key = `Name` (= Party Name / Account Name). Provides `State`, `Station`, `Group Name` (zone) per party. Also per-FY.

### `ReadMe`
Documentation tab (skip for parsing). Header row ~index 9. Documents the FY periods and per-sheet row counts.

### Dates — IMPORTANT GOTCHA
With `xlsx.readFile(file,{cellDates:true})`, `Bill Date` comes back as a JS `Date` but shifted by the IST offset, e.g. first FY24-25 sale shows `"2024-03-31T18:29:50.000Z"` which is **2024-04-01 in IST (+5:30)**. So **do not** use `getUTCMonth()` naively — you'll land in the wrong month/FY. Recommended: derive the month key from `Financial Year` + the date's IST-local components, OR add the IST offset before extracting Y/M. **Prefer the `Financial Year` column as the source of truth for the year; use the date only for the month-within-FY.** Verify against how the OLD `dashboardBuilder.js` currently derives `month` (it parses the old workbook's dates similarly).

## 7. GOTCHAS / RULES

- **Restart the node server after any `server/*.js` edit** (no hot reload). This bit us twice.
- **IST date offset** (§4) — get month/FY right or all aggregates shift.
- **`Amount Grand Total` is bill-level** — only sum it on header rows (`isHeader`) for party/bill totals; use line-level `Amount` for item breakdowns. Mirror how the old builder split `finalAmount` vs `amount`.
- Per-FY masters (ItemMaster/AccountMaster have 3 rows per entity) — dedupe by name (latest FY) or join per-FY.
- Keep the public chart component APIs unchanged (`BarChart rows`, `DonutChart rows`, `LineChart series/months/labels`) so pages don't break.
- No em-dashes rule is a content preference, not code. `money()` must stay `₹… Lakh`.
- bun/TypeScript preferences in global CLAUDE.md do NOT apply here — this project is plain JS + npm; match existing style.
- Don't re-introduce Supabase or a database. Stateless, direct-to-sheet is the architecture.
